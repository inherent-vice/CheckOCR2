## 2024-05-18 - DataFrame Iteration in Excel Import
**Learning:** Using `df.iterrows()` when iterating over Pandas DataFrames creates significant overhead, taking ~2.1 seconds for 10,000 rows. Converting the DataFrame to a list of dicts with `df.to_dict('records')` before iterating, and caching column names upfront, drastically improves performance (takes ~0.64 seconds for the same task). This is because `iterrows()` yields a Series object for each row, adding considerable memory and object creation overhead.
**Action:** When performing row-by-row iteration on Pandas DataFrames in performance-sensitive parts of the application, always prefer `df.to_dict('records')` or `df.itertuples()` over `df.iterrows()`. In CheckOCR2, applying this inside `checkocr2/excel_io.py` can drastically speed up Excel file loading times.
